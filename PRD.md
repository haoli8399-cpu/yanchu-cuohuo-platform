## 1. 产品战略定位

### 1.1 竞争策略

**喜剧工厂是「AI驱动的商演交易平台」**，不是纯撮合平台，也不是纯AI工具。

| 维度 | 撮合平台 | AI经纪人工具 | 喜剧工厂（融合模式） |
|:----:|:--------:|:-----------:|:-------------------:|
| 供给端 | 需要地推 | 不需要 | ✅ 已有艺人/SKU库 |
| 需求端 | 需要获客 | 获客成本高 | ✅ 已有活动公司 |
| 交易效率 | 人工匹配 | AI自动 | ✅ AI辅助+人工确认 |
| 壁垒 | 网络效应 | 数据+AI | ✅ 网络+AI双重壁垒 |
| 规模化 | 重运营 | 轻模式 | ✅ 逐步降低运营成本 |

### 1.2 核心差异化

> **不是帮用户「管理演出」，而是帮用户「把商演需求变成成交订单」。**

- 竞争对手（活动行、大麦）：演出信息平台，交易在站外
- 我们：商演交易平台，从询价到签约到收款全闭环
- 终局：AI经纪人接管80%标准化工作，人做20%复杂决策

## 2. 产品一句话定义

> **AI驱动的B2B商演交易平台——帮助活动公司15分钟完成从「有个需求」到「拿到报价方案」的全过程，帮助艺人/厂牌从零散微信询价走向标准化、可跟进的订单管理。**

## 3. 产品不是什么

| 不是 | 为什么 |
|:----|:----|
| ❌ **不是纯招投标平台** | 活动公司不应该"几家公司竞价"，而是平台基于需求标准化匹配最优方案 |
| ❌ **不是大众点评/大麦网** | 核心用户是B端活动公司，不是C端消费者 |
| ❌ **不是艺人管理系统** | 艺人管理是底层数据，不是产品主入口 |
| ❌ **不是全自动AI** | 首版坚持：AI建议+人确认，不做AI自动对外发送 |
| ❌ **不是全行业平台** | 只聚焦脱口秀/喜剧/小剧场商演，不做演唱会/明星经纪 |

## 4. 目标用户与核心场景

### 4.1 用户角色

最终角色清单（7类）：

| 角色 | 归属于 | 使用端 | 核心功能 |
|:----|:------|:------|:--------|
| 🏢 活动公司 | 演艺电商 | 小程序 + PC Web | 浏览方案、比价、提交需求、查看渠道价 |
| 🏢 直接甲方 | 演艺电商 | 小程序 + PC Web | 同上，但看到的是甲方标准价 |
| 🔧 平台管理员 | 演艺电商 | platform-admin (PC) | 供应商管理、佣金设置、全平台看板、仲裁 |
| 🎯 自营运营 | 演艺电商 | supplier-console (PC) | 接自营订单、管艺人、SKU管理、看利润 |
| 🏪 第三方经纪公司 | 演艺电商(平台订单) + Agent(自有客户) | supplier-console (PC) | 平台接单 + Agent处理自有客户 |
| 🎤 独立艺人 | 演艺电商(平台订单) + Agent(自有客户) | supplier-console小程序版 + 艺人模式 | 接单、管SKU、看收入 |
| 🎭 经纪公司旗下艺人 | AI商演经纪人（不含电商平台） | 小程序艺人模式 | 排期、签到、结算、信誉 |

**角色关系说明：**

- **活动公司与直接甲方是两个不同角色**，核心区别：活动公司看到渠道价，直接甲方看到甲方标准价
- **经纪公司旗下艺人不直接出现在演艺电商平台**——他们由经纪公司在 Agent 内管理，仅通过小程序艺人模式查看排期、签到、结算
- **独立艺人是特殊角色**：既是 supplier（管理自己SKU、接单）又是 performer（排期、演出、签到、结算）。两个模式都能用
- **自营运营、第三方经纪公司共用同一套 supplier-console**（PC为主，小程序做核心操作补充）
- **平台管理员使用独立的 platform-admin**
- 小程序提供两个角色入口：
  - **活动公司/直接甲方模式**：默认 tab 列（发现 | 方案 | 提需求 | 消息 | 我的）
  - **艺人模式**：切换后显示排期/结算/签到 tab

### 4.2 核心用户场景

**场景一：活动公司年会找脱口秀（已有）**

```
HR打开小程序 → 
  方式A: 浏览SKU列表，点「脱口秀标准版」，看详情和渠道价格，点「获取报价」
  方式B: 输入/粘贴需求文字"年会10月20日，300人，要脱口秀，预算1万以内"
        → AI推荐方案+估价 → 确认提交

提交后 → 运营工作台收到任务 → AI预生成方案+推荐艺人 
       → 运营确认/调整 → 点击发送 → 活动公司微信收到方案通知
       → 活动公司确认 → 签约 → 付款 → 演出现场签到 → 评价
```

> 注：直接甲方的流程同上，区别在于看到的SKU价格为甲方标准价而非渠道价。

**场景二：艺人管理排期（已有）**

```
艺人打开小程序 → 看到今日排期 → 看到待确认任务 → 
确认演出 → 当天签到打卡 → 演出后进入结算流程 → 团队收到款 → 信誉分更新
```

**场景三：自营运营处理需求（新增）**

```
自营运营打开 supplier-console (PC) → 
工作台展示新需求卡片 + AI预生成方案 →
  查看AI推荐的SKU+艺人+报价 →
  调整（换艺人级别/改时长/调价格） →
  点击「发送方案」→ 活动公司收到通知 →
  跟踪确认状态 → 活动公司确认后安排艺人 → 跟进演出执行
```

**场景四：第三方经纪公司接单（新增）**

```
第三方经纪公司打开 supplier-console (PC) → 
工作台收到平台分配的需求 →
  查看需求详情（人数/预算/日期/活动类型） →
  基于平台推荐报价，自主报价（可调整） →
  安排旗下艺人（选人+定档期） →
  提交方案 → 活动公司端收到方案 →
  确认后签约 → 演出执行 → 结算 → 平台抽佣
```

**场景五：独立艺人管理自己（新增）**

```
独立艺人打开小程序，切换到「艺人模式」→ 
  查看排期日历 + 待确认演出 →
  
同时打开 supplier-console 小程序版 →
  管理个人SKU（价格/时长/类型/可演出日期） →
  查看历史订单和收入 →
  收到平台订单 → 确认/拒绝 →
  演出 → 签到 → 结算 → 收入到账
```

## 6. 核心业务流程

### 6.1 活动公司端（完整链路）

```
发现方案 → 查看详情 → 提交需求 → 收到报价 → 确认方案 → 
签约 → 等待演出 → 演出执行 → 评价
```

| 步骤 | 页面 | 功能 |
|:----:|:----:|------|
| 1 | 发现页 `discover` | 搜索/Banner/分类/推荐 |
| 2 | SKU详情 `sku/detail` | 默认配置+价格+介绍+场地+案例+评价 |
| 3 | 提交需求 `request/submit` | 自定义配置器（级别/时长/人数）+ 实时报价 |
| 4 | 需求列表 `request/list` | 状态跟踪 |
| 5 | 需求详情 `request/detail` | AI方案+确认/不采纳+评价 |
| 6 | 个人中心 `user` | 收藏/订单/企业信息/设置/通知 |

> 注：直接甲方的流程同上，唯一区别是SKU中展示的价格为甲方标准价。

### 6.2 艺人端

```
收到排期 → 确认档期 → 现场签到 → 结算 → 信誉更新
```

### 6.3 运营端

```
新需求通知 → 查看需求 → AI辅助生成方案 → 调整/确认 → 发送
→ 跟踪确认 → 安排艺人 → 跟进执行 → 完成归档
```

## 7. 商业模式

### 7.1 模型定位：京东自营模型

> 喜剧工厂借鉴京东自营模式——平台自己是市场管理者，"喜剧工厂自营"是平台上一家供应商。活动公司在方案列表中看到的是「混排」结果：自营方案和第三方供应商方案并列展示，通过供应商标识区分。

### 7.2 三层渐进收入模型

#### Tier 1：自营赚差价（当前阶段）

- **模式：** 喜剧工厂签约艺人，作为唯一供应商
- **盈利方式：** 赚取演出差价（活动公司付¥6,000 → 艺人出场费¥4,000 → 平台利润¥2,000）
- **目标毛利率：** 30-40%
- **启动条件：** 现在（已有艺人库和SKU体系）
- **关键动作：** 定价策略 + 运营效率提升

#### Tier 2：平台抽佣（月需求≥100单后开放）

- **模式：** 第三方经纪公司/独立艺人入驻，与自营混排展示
- **盈利方式：**
  - 平台带来的订单 → 抽佣 **10-15%**（按订单金额计算）
  - 第三方自己带来的订单 → **不抽佣**
- **目标佣金率：** 12%
- **启动条件：** 月需求≥100单（约 Phase 2 中后期）
- **关键动作：** 供应商入驻流程 + 佣金结算系统 + 品质管控

#### Tier 3：AI Agent 订阅（Agent 打磨成熟后开放）

- **模式：** 第三方经纪公司/独立艺人可订阅 AI 商演经纪人处理商演邀约
- **定价：**
  - 个人版：**¥199/月**（独立艺人用）
  - 企业版：**¥499/月**（经纪公司用）
- **核心功能：** 商机判断、AI 报价、回复话术生成、智能跟进提醒、合同生成、谈判建议
- **规则：** 订阅用户的自有渠道订单 → 0 佣金（订阅费已覆盖）
- **启动条件：** AI Agent 功能成熟、供应商量达到 10+ 家
- **关键动作：** Agent 产品化 + 订阅计费系统 + 非平台来源订单管理

### 7.3 三层收入关系

三层模型不是替代关系，而是叠加关系：

| 收入来源 | Tier 1 | Tier 2 | Tier 3 |
|:--------|:-----:|:-----:|:-----:|
| 自营差价 | ✅ | ✅ | ✅ |
| 平台佣金 | — | ✅ | ✅ |
| AI Agent 订阅 | — | — | ✅ |

**逐步演进示意：**

```
Tier 1 阶段（当前）
  收入 = 自营差价
  月营收目标：¥6万（假设月订单10-15单，均价¥6,000，毛利40%）

Tier 1+2 阶段（Phase 2 中后期）
  收入 = 自营差价 + 平台佣金
  自营：保持不变
  第三方：假设月订单20单，均价¥5,000，佣金12%
  月营收目标：自营¥6万 + 佣金¥1.2万 = ¥7.2万

Tier 1+2+3 阶段（Phase 3）
  收入 = 自营差价 + 平台佣金 + Agent订阅
  自营：¥6万
  第三方佣金：¥3万（月订单50单，均价¥5,000，佣金12%）
  Agent订阅：¥7,500（5家公司×¥499 + 20个艺人×¥199）
  月营收目标：¥9.75万
```

### 7.4 价格体系

明确三层价格：

| 价格类型 | 谁看到 | 含义 | 计算方式 |
|:--------|:-----|:----|:------|
| 甲方标准价 | 直接甲方、所有人可见的公开价 | 对外统一报价 | SKU中配置 |
| 活动公司渠道价 | 活动公司 | 分销渠道成本价 | 标准价 × 折扣系数（0.7） |
| 艺人成本 | 仅平台运营+管理员 | 内部结算 | 手动输入 |

**价格展示规则：**

- 登录后根据角色显示对应价格
- **活动公司登录** → SKU列表/详情显示渠道价，小字提示"甲方标准价 ¥X,XXX"
- **直接甲方登录** → SKU列表/详情显示甲方标准价
- **提交需求页配置器** → 根据登录角色取对应的价格基础计算（渠道价或标准价）
- **未登录** → 显示甲方标准价（公开价）

**第三方价格：**

- 第三方供应商自行定价，不受平台标准价体系约束
- 第三方SKU展示价格 = 供应商自主设定价格
- 平台在此基础上收取佣金 12%

**已有价格体系补充（兼容不改）：**

| 价格类型 | 说明 | 适用对象 |
|:--------|:----|:--------|
| 甲方标准价 | 活动公司端展示的对外报价 | 直接甲方（公开价） |
| 活动公司渠道价 | 活动公司的成本价（= 标准价 × 0.7） | 活动公司 |
| 内部结算价 | 平台给艺人的出场费 | 自营艺人 |
| 第三方自主定价 | 第三方供应商自行定价 | 第三方订单 |
| Agent 订阅费 | 个人¥199/月，企业¥499/月 | Tier 3 订阅用户 |

#### 10.2.3 AI 主动追问机制

**必须追问，否则就不是AI经纪人。** 真实场景中活动公司HR通常只写一句模糊需求（"年会想搞个脱口秀"），不追问无法匹配方案。

**追问策略：**

```
用户: "年会想搞个脱口秀"
         ↓
AI: 好的！我先确认几个信息：
    ① 多少人参加？
      [50-100人] [100-300人] [300-500人] [500人以上]

用户: 选 100-300人
         ↓
AI: ② 预算范围？
      [5千以内] [5千-1万] [1万-2万] [2万以上] [还没想好]

用户: 1-2万
         ↓
AI: ③ 大概什么时间？
      [请选择日期]

用户: 选 10月20日
         ↓
AI: 明白了——300人年会，10月20日，预算1-2万。
    推荐「脱口秀标准版」T3级 60min ¥6,000 →
    再加一个备选「脱口秀旗舰版」T2级 90min ¥9,000 →
    [确认提交]
```

**追问规则：**

| 缺失字段 | 追问方式 | 是否必填 |
|:--------|:--------|:--------:|
| 人数 | 提供几个常见范围按钮（快速选择） | 是（影响艺人配置） |
| 日期 | 日期选择器 | 是（影响档期匹配） |
| 预算 | 几个价格区间按钮 | 是（影响SKU推荐） |
| 地点 | 输入框 | 否（可选，辅助场地匹配） |
| 特殊要求 | 输入框 | 否（所有其他要求写在这里） |

**追问次数上限：最多3轮。3轮后使用已有信息直接匹配。** 避免把用户困在问卷里。

**追问状态指示：**

```
追问中（如"①/③ 还需补充2个信息"）
追问完成 → 自动开始匹配
匹配中（如"🤖 正在为你匹配方案..."）→ 展示结果卡片
```

#### 10.2.4 AI 自然语言识别

**输入：** 自由文本（上限500字）

**识别字段：**

| 字段 | 识别方式 | 示例 |
|:----|:--------|:----|
| 演出类型 | 关键词匹配 + 语义理解 | "脱口秀""即兴""魔术""喜剧" |
| 活动类型 | 上下文判断 | "年会""团建""发布会""开盘" |
| 人数规模 | 数字提取 | 300人 |
| 预算范围 | 数字+语义 | "1万以内""5千-8千" |
| 日期/时段 | 日期识别 | 10月20日/下个月初 |
| 地点 | 地名识别 | 上海浦东 |
| 特殊要求 | 语义理解 | "要女艺人""最好会互动""不要太黄段子" |

#### 10.2.5 推荐匹配逻辑

```
输入文字 → AI 结构化 → 
  Step 1: 匹配演出类型 → 过滤SKU库
  Step 2: 匹配预算 → 过滤价格区间
  Step 3: 匹配人数 → 推荐艺人人数
```

### 11.3 Supplier Console — 基础功能 vs Agent订阅

supplier-console 的功能分为两部分：基础功能（所有供应商都有）和 Agent 订阅功能（Tier3 订阅后解锁）。

#### 11.3.1 基础功能（所有供应商免费使用）

| 模块 | 功能 | 自营 | 第三方经纪公司 | 独立艺人 |
|:----:|------|:----:|:------------:|:-------:|
| 工作台 | 待处理平台订单 + 数据概览 | ✅ | ✅ | ✅ 精简 |
| 订单管理 | 接单→报价→安排艺人→跟踪 | ✅ | ✅ | ✅ |
| SKU管理 | 个人/公司SKU模板配置 | ✅ | ✅ | ✅ |
| 艺人管理 | 旗下艺人CRUD+排期 | ✅ | ✅ | — |
| 结算看板 | 待结算/已结算+佣金明细+利润 | ✅ | ✅ | ✅ |
| 利润看板 | 毛利/净利润 + 趋势图 | ✅ | — | — |

#### 11.3.2 Agent 订阅功能（Tier3 订阅后解锁）

以下功能仅对订阅了 AI Agent（个人版 ¥199/月 或 企业版 ¥499/月）的供应商开放：

| 能力 | 描述 | 示例 |
|:----|:----|:----|
| **AI商机判断** | 分析客户询价文本，判断是否值得接 | "这个客户预算太低，建议放弃" |
| **AI报价生成** | 基于艺人级别、时长、人数、历史行情自动生成报价 | "推荐报价¥6,000（同类案例均价¥5,500-7,500）" |
| **AI回复话术** | 生成面对不同场景的回复文案 | 客户嫌贵→"建议您可以选择T4级方案..." |
| **AI谈判建议** | 客户有异议时给出谈判策略和议价空间 | "建议还价空间¥500-1,000" |
| **智能跟进提醒** | N天未确认自动推送跟进提醒 | "方案已发3天未确认，建议跟进" |
| **AI合同生成** | 方案确认后生成合同草稿 | 自动填充演出方/需求方/价格/日期/条款 |
| **自有渠道订单管理** | 非平台来源客户的订单（客户管理+沟通记录+成交历史） | 独立于平台订单的客户CRM模块 |

**关键原则：**

> - AI 负责判断、建议、生成、提醒
> - 人负责确认、发送、决策
> - 不替代人，而是让一个新人经纪人也能按标准SOP完成商务流程

#### 11.3.3 supplier-console 内部结构

```
┌─ supplier-console ──────────────────────────┐
│  [平台订单] [自有订单] │ SKU管理 │ 艺人管理   │
│  ↑ 基础功能 ↑ Agent解锁 │          │          │
│                                              │
│  如果订阅Agent → 所有Tab显示AI辅助能力        │
│  如果未订阅 → 仅基础功能，无AI提示             │
└──────────────────────────────────────────────┘
```

- **平台订单 Tab**（基础功能）：来自平台的匹配需求，接单→报价→安排艺人→发送方案
- **自有订单 Tab**（Agent 解锁）：供应商自有渠道客户的订单管理，AI 辅助全程
- **SKU管理 Tab**（基础功能）：方案配置+价格+类型，Agent 订阅后可获得 AI 定价建议
- **艺人管理 Tab**（基础功能）：旗下艺人CRUD+排期+档期，Agent 订阅后可获得 AI 艺人匹配推荐

#### 11.3.4 部署

- PC Web端：`/supplier/` 子路径（与 `/admin/` 对称）
- 小程序端：供应商核心操作（接单确认、排期查看）通过小程序供应商视图完成
- 技术栈：待定（推荐与 admin 相同的 Umi 4 + Ant Design 技术栈以降低维护成本）

### 11.4 AI Agent 订阅制

#### 11.4.0 AI 商演经纪人定位

> **AI Agent 不是「自有客户 CRM」或「智能获客工具」，而是面向供应商的 AI 商演经纪人——负责帮经纪公司/独立艺人把商演邀约转化为成交订单。**

**核心能力（参考《AI 商演经纪人 Agent》PRD）：**

```
客户发来一句话 → AI 立刻帮你判断、报价、回复、跟进、签约。
```

| 能力 | 描述 | 示例 |
|:----|:----|:----|
| **商机判断** | 分析客户询价文本，判断是否值得接 | "这个客户预算太低，建议放弃" |
| **AI 报价** | 基于艺人级别、时长、人数、历史行情自动生成报价 | "推荐报价¥6,000（同类案例均价¥5,500-7,500）" |
| **回复话术** | 生成面对不同场景的回复文案 | 客户嫌贵→"建议您可以选择T4级方案..." |
| **谈判建议** | 客户有异议时给出谈判策略和议价空间 | "建议还价空间¥500-1,000" |
| **跟进提醒** | 自动提醒跟进节点 | "方案已发3天未确认，建议跟进" |
| **合同生成** | 方案确认后输出合同草稿 | 自动填充演出方/需求方/价格/日期/条款 |

**关键原则（与附件PRD一致）：**

> - AI 负责判断、建议、生成、提醒
> - 人负责确认、发送、决策
> - 不替代人，而是让一个新人经纪人也能按标准SOP完成商务流程

#### 11.4.1 订阅套餐

| 套餐 | 价格 | 适用对象 | 包含功能 |
|:----:|:----:|:--------|:--------|
| **免费版** | ¥0 | 所有供应商 | 浏览SKU、手动提交需求、等待运营匹配 |
| **个人版** | ¥199/月 | 独立艺人 | AI自动报价 + 智能方案推荐 + 3次跟进提醒 + 商务邀约处理 |
| **企业版** | ¥499/月 | 经纪公司（含自营） | 个人版全部功能 + 无限跟进提醒 + 合同生成 + 完整经纪人工作台 + 历史报价对比 |

#### 11.4.2 计费规则

- 免费试用期 14 天
- 按月订阅，自动续费，支持随时取消
- 订阅用户的自有渠道订单 → **0 佣金**（订阅费已覆盖平台服务成本）
- 平台带来的订单 → 正常抽佣 12%

#### 11.4.3 Agent 核心能力

| 能力 | 说明 | 使用方式 |
|:----|:----|:--------|
| **AI 报价** | 基于历史成交价+客户类型+活动类型+紧迫度的智能报价 | supplier-console 内置 |
| **智能跟进** | 自动识别未确认/未签约状态，定时推送提醒 | 自动触发 + 可配置 |
| **合同生成** | AI 基于方案信息自动生成合同草稿，供应商审阅后发送 | supplier-console → 一键生成 |
| **自有渠道订单管理** | 供应商管理自己的客户（非平台来源），跟踪沟通记录和成交历史 | supplier-console 独立模块（Agent解锁） |

### 11.5 在线签约 + 支付闭环

#### 11.5.1 签约流程

```
方案确认 → AI 生成合同草稿 →
  供应商审阅/编辑 →
  活动公司查看/签署 →
  供应商签署 →
  合同归档 → 进入支付流程
```

#### 11.5.2 支付流程

```
合同签署 → 活动公司支付定金（50%）→
  艺人安排 → 演出执行 → 活动公司确认完成 →
  支付尾款（50%）→ 供应商结算
```

#### 11.5.3 资金流向

| 订单类型 | 资金流向 | 平台收费 |
|:--------|:--------|:--------|
| 自营订单 | 活动公司 → 平台 → 7天后 → 艺人 | 差价（毛利率 30-40%） |
| 第三方+平台订单 | 活动公司 → 平台 → 7天后 → 供应商 | 佣金 12% |
| 第三方+自有渠道 | 活动公司 → 供应商（平台不托管） | 0%（订阅已覆盖） |

### 11.6 智能报价引擎全面开放

**报价逻辑：**

```
活动公司询价 → AI分析：
  ① 客户类型（首次/老客户）
  ② 活动类型（年会预算通常高于团建）
  ③ 时间紧迫度（紧急需求可溢价）
  ④ 地理位置（一线城市可溢价）
  ⑤ 历史成交价（同类案例的成交价分布）
  ⑥ 供应商类型（自营/第三方/独立艺人）
  → 输出推荐报价区间
```

**价格区间说明：**

| 价格点 | 含义 | 示例（T3级 60分钟 2人） |
|:------:|------|:----------------------:|
| 底线价 | 低于此价不接 | ¥4,800 |
| 推荐价 | 基于历史数据的最佳成交价 | ¥6,000 |
| 目标价 | 可争取的较高价 | ¥7,500 |
| 上限价 | 不应超过的合理价格 | ¥9,000 |

## 12. 技术架构与API规范

### 12.1 现有技术栈

| 层 | 技术 | 版本 |
|:--:|:----|:----:|
| 小程序前端 | uni-app + Vue 3 + Vant Weapp | 最新 |
| PC Web端 | Vite + React 18 + Ant Design 6 | 最新 |
| 运营后台 | Umi 4 + @umijs/max + ProTable | 4.6.69 |
| API | Fastify + TypeScript + Zod | 4.x |
| 数据库 | PostgreSQL | 15 |
| 部署 | 腾讯云香港 2C2G Ubuntu 22.04 | — |

### 12.2 API 设计原则

1. **RESTful：** 所有端点以 `/v1/` 开头
2. **Zod 校验：** 所有请求参数使用 Zod Schema 校验
3. **分页标准：** `page: 1-*, pageSize: 1-1000, 返回 paginatedResponse`
4. **错误码统一：** `0` 成功 / `1xxx` 认证 / `99xx` 参数 / `9999` 服务器错误
5. **JWT 认证：** 所有需要认证的端点使用 `authMiddleware`
6. **API Contract：** 所有 API 变更必须同步更新 `docs/API_CONTRACT.md`

### 12.3 AI 服务接口（Phase 2+）

```
POST /v1/ai/parse-demand
  输入：{ text: string }
  输出：{ demand: { type, activity_type, crowd_size, budget_range, date, location, requirements } }

POST /v1/ai/recommend-plan
  输入：{ demand_id: string }
  输出：{ plans: [{ sku_id, tier, duration, performer_count, price, confidence, reasoning }] }

POST /v1/ai/recommend-price
  输入：{ sku_id, tier, duration, crowd_size, activity_type, location, urgency }
  输出：{ base_price, recommended_price, target_price, ceiling_price, reasoning }
```
