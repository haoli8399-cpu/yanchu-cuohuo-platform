# 迭代产出归档

> 每次迭代的完整产出存档，供追溯和回滚。
> 由 Orchestrator 在每轮任务完成后写入。
> Developer / Tester 只读不改。

---

## 目录结构

```
rev/
├── rNN/              # 第 N 次迭代（从 r01 开始递增）
│   ├── TASK.md      # 当时派发的任务文件（拷贝）  
│   ├── output.md    # 产出说明（改了啥文件、代码概要）
│   └── test.md      # 测试记录（Tester 的功能验证结果）
├── rNN/
│   └── ...
└── README.md         # 本文件
```

## 归档规则

| # | 规则 | 说明 |
|:-:|------|------|
| 1 | **任务完成后归档** | OC 审查通过后、同步文档前，拷贝 TASK.md 到 `rev/rNN/TASK.md` |
| 2 | **编号递增** | 从 `r01` 开始，每轮 +1，不跳号 |
| 3 | **output.md 为 OC 写** | 变更说明 + 文件清单 + 涉及决策 |
| 4 | **test.md 为 Tester 写** | 功能验证结果 + 问题清单（如有） |
| 5 | **不删不改历史** | 已归档的迭代不能删除或修改，只能追加新迭代 |
