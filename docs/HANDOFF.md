## 四、核心业务流程（30秒理解）

```
用户（选SKU或提需求）→ AI出方案 → 运营审核调整 → 活动公司确认
  → 签约（跳过/上传PDF/电子签）→ 线下定金 → 运营标记定金已收
  → 运营锁定演员 → 演员确认排期
  → 演出 → 签到打卡（计入信誉分）
  → 验收 → 线下尾款 → 运营标记尾款已收
  → 线下打款给演员 → 运营标记已结算
  → 双方评价（48h双向+超时默认4星）
```

## 六、关键设计决策

| 决策 | 结论 | 原因 |
|------|------|------|
| 不做在线支付 | ✅ 改为线下登记 | 前期交易量小，后仰现有打款流程可支撑 |
| 不做竞标模式 | ✅ | 冷启动死锁 |
| 不做活动公司零售价管理 | ✅ | 商业机密 |
| SKU显示演员画像非锁定阵容 | ✅ | 避免用户误解演员固定 |
| 活动公司两步入驻 | ✅ | 注册即用，交易前企业认证挡板 |
| 活动公司端 | ✅ Web端完整版+小程序基础版 | Web端承载多订单/对比/导出 |
| AI方案保留 | ✅ 改为SKU匹配+微调 | 演出非标品，需要定制感 |
|| 结算降级为统计 | ✅ 线下操作+系统记录 | 减少MVP复杂度 |
|| 演员信誉分四维度 | ✅ 履约/质量/活跃/基础 | 科学可量化 |
|| 运营后台RBAC分权限 | ✅ 超级管理员/运营/财务/内容 四种角色 | 敏感操作隔离+审计合规 |
