# PRD功能 · 基于新定位的优化方案

## 新定位

**不是"批发商"，也不是"撮合平台"——是"演出供应链保障平台"。**

核心差异：平台不只是交易渠道，更重要的角色是**履约保障方**。

## 价值主张链

```
    平台（保障方）
    ┌──────────────────────────────┐
    │ 保供给：留住优质演员          │
    │ 保需求：吸引更多活动公司      │
    │ 保履约：兜底质量/放鸽子/坏账  │
    └──────────┬───────────────────┘
               │
     ┌─────────┴──────────┐
     ▼                    ▼
  演员（供给侧）         活动公司（需求侧）
  ├ 更多活儿             ├ 更多方案（行业经验积累）
  ├ 信息积累（评价+案例）  ├ 一手靠谱演员
  └ 演出费零忧虑（平台兜底）└ 履约零忧虑（平台兜底）
```

## 功能优化方向（按价值主张驱动）

### 针对演员：多接活 + 信息积累 + 费无虑

| # | 功能 | 优化方向 | 对应价值 |
|:-:|------|---------|---------|
| B-01 | 首页卡片 | 不变 | 多接活 |
| B-02 | 排期确认 | 不变 | 多接活 |
| B-03 | 价格协商 | 简化，固定批发价为主 | — |
| B-09 | 结算中心 | **升级**：展示"即时到账48h"进度条+兜底说明 | **费无虑** |
| B-11 | 我的主页 | **升级**：作为演员的"职业信用档案"，沉淀评价+案例+信誉分 | **信息积累** |
| B-07 | 阵容查看 | **升级**：突出跟谁同台的信息价值（演员社交网络） | 信息积累 |
| B-08 | 演出反馈 | **升级**：反馈作为信誉分的组成部分，直接影响推荐优先级 | **信息积累** |
| 新增 | 信誉分体系 | 演员信用分（接单率+评价+演出质量），高分优先派单 | **多接活+信息积累** |

| A-12 | 历史记录 | **升级**：展示用过的演员、效果评价，下次可直接复用 | **演员靠谱** |
| A-16 | 指定演员 | 不变 | — |
| A-13 | 付款 | **升级**：突出平台兜底（"演员演砸了平台赔"） | **履约无忧** |
| A-10 | 合同管理 | 不变 | — |
| A-11 | 方案导出Word | 不变 | — |
| 新增 | **活动公司信誉分** | 活动公司信用分（付款准时率+合作次数），高分可获优先服务 | 供给侧管理 |

### 运营后台（平台）

| # | 功能 | 优化方向 | 对应价值 |

| P-09 | 排期分配 | 不变 | — |

## 一句话总结

**功能不减，方向微调。核心差异是：平台从"交易工具"升级为"履约保障方"，所有功能的优化都围绕三个价值主张——演员多接活+信息积累+费无虑，活动公司方案多+演员靠谱+履约无忧。**
