## 四、数据库设计（11张核心表）

### 4.1 用户相关

```sql
-- 用户表（统一认证）
users (
  id UUID PK,
  phone VARCHAR(11),
  wechat_openid VARCHAR,           -- 微信openid（用于推送+扫码登录）
  name VARCHAR,
  avatar_url TEXT,
  role ENUM('client','agent','performer','admin'),
  created_at TIMESTAMP
)

-- 活动公司资料表
company_profiles (
  id UUID PK,
  user_id FK → users,
  status ENUM('registered','pending_cert','certified'),  -- 注册/待认证/已认证
  short_name VARCHAR,               -- 公司简称
  full_name VARCHAR,                -- 公司全称（认证后填写）
  credit_code VARCHAR,              -- 统一社会信用代码
  business_license_url TEXT,        -- 营业执照扫描件
  legal_person_name VARCHAR,        -- 法人姓名
  legal_person_id_url TEXT,         -- 法人身份证照片
  bank_name VARCHAR,                -- 开户银行
  bank_account VARCHAR,             -- 对公账号
  contact_person VARCHAR,           -- 联系人
  city VARCHAR,
  service_categories JSONB,         -- 服务类目
  created_at TIMESTAMP
)

-- 演员资料表（含展示信息）
performers (
  id UUID PK,
  user_id FK → users,
  name VARCHAR,
  phone VARCHAR(11),
  avatar_url TEXT,
  cover_url TEXT,                   -- 封面大图（杂志风）
  style_tags JSONB,                 -- 风格标签
  introduction TEXT,                -- 个人简介
  highlights TEXT,                  -- 个人亮点
  media_urls JSONB,                 -- 作品视频/照片
  social_links JSONB,               -- 社交媒体
  experience_years INT,             -- 从业年限
  rating DECIMAL(2,1),             -- 综合评分
  credit_score DECIMAL(4,2),       -- 信誉分
  credit_level ENUM('S','A','B','C','D'),  -- 信誉等级
  tier ENUM('T0','T1','T2','T3','T4','T5','T6'),  -- 咖位等级
  tier_updated_at TIMESTAMP,         -- 咖位最后更新时间
  tier_updated_by FK → users,        -- 咖位操作人
  status ENUM('active','inactive'),
  created_at TIMESTAMP
)
```

### 4.2 核心业务

```sql
-- SKU方案表（标准化演出产品）
skus (
  id UUID PK,
  name VARCHAR,
  description TEXT,
  performer_profile TEXT,            -- 演员画像（非锁定阵容）
  style_tags JSONB,                  -- 适用风格
  applicable_scenes JSONB,           -- 适用场景（年会/开业/派对等）
  base_price DECIMAL(12,2),         -- 批发价
  duration_minutes INT,              -- 标准时长
  performers_count INT,              -- 演员数量
  media_urls JSONB,                  -- 样片/案例
  status ENUM('active','inactive'),
  created_at TIMESTAMP
)

-- 三种价格配置表（运营后台维护）
price_configs (
  id UUID PK,
  config_type ENUM('performer_settlement','agent_quote','client_quote'),
  business_line ENUM('venue_booking','outdoor_show','show_sponsor','custom_content','custom_showcase'),
  tier ENUM('T0','T1','T2','T3','T4','T5','T6'),  -- 仅内部结算价使用
  package_name VARCHAR,              -- 套餐名（基础版/标准版/加长版/豪华版）
  duration_minutes INT,              -- 时长
  performer_count INT,               -- 标配人数
  base_price DECIMAL(12,2),         -- 基准价
  agent_discount DECIMAL(3,2) DEFAULT 0.7,  -- 活动公司折扣（全局）
  extra_fee_per_5min DECIMAL(12,2), -- 每+5min加价（内部结算）
  script_creation_fee DECIMAL(12,2),-- 定制段子创作费/分钟
  script_performance_fee DECIMAL(12,2), -- 定制段子表演费/分钟
  remote_fee_in_city DECIMAL(12,2), -- 绕城外附加费
  remote_fee_out_city DECIMAL(12,2), -- 成都市外附加费
  updated_by FK → users,             -- 最后修改人
  updated_at TIMESTAMP,
  created_at TIMESTAMP
)

-- 方案需求表
demands (
  id UUID PK,
  client_id FK → users,              -- 活动公司/甲方
  business_line ENUM('direct','agent'),
  source ENUM('sku','requirement','phone'),  -- SKU选购/需求提报/电话代客
  sku_id FK → skus,                 -- 关联SKU（选购路径）
  title VARCHAR,
  event_type VARCHAR,
  event_date DATE,
  event_time TIME,
  city VARCHAR,
  address TEXT,
  budget DECIMAL(12,2),
  audience_count INT,
  ai_plan_content TEXT,              -- AI方案内容
  adjusted_plan_content TEXT,        -- 运营调整后的方案
  final_plan_content TEXT,           -- 最终确认方案
  final_price DECIMAL(12,2),
  urgency ENUM('normal','urgent','emergency'),
  contract_mode ENUM('skip','upload','system'),  -- 签约模式
  operator_id FK → users,
  status ENUM(                       -- 12个状态
    'pending_ai','ai_generated','pending_operator','operator_adjusted',
    'pending_client_confirm','confirmed','pending_deposit','deposit_received',
    'pending_performer','performer_confirmed','performing','finished',
    'pending_final_payment','final_payment_received','settled',
    'cancelled','refunding'
  ),
  created_at TIMESTAMP,
  updated_at TIMESTAMP
)

-- 排期分配表
assignments (
  id UUID PK,
  demand_id FK → demands,
  performer_id FK → performers,
  performance_role VARCHAR,          -- 角色（脱口秀/即兴/主持等）
  arrival_time TIMESTAMP,            -- 到场时间（早于开演30-60分钟）
  checkin_time TIMESTAMP,            -- 签到打卡时间
  checkin_location JSONB,            -- 签到位置
  negotiated_price DECIMAL(12,2),
  status ENUM('pending','confirmed','rejected','completed','cancelled'),
  reject_reason VARCHAR,
  confirmed_at TIMESTAMP,
  created_at TIMESTAMP
)

-- 阵容表（多演员同场）
lineup (
  id UUID PK,
  demand_id FK → demands,
  performer_id FK → performers,
  role VARCHAR,
  status ENUM('pending','confirmed','rejected'),
  created_at TIMESTAMP
)
```

## 六、状态机（12个状态）

```
pending_ai ──► ai_generated ──► pending_operator ──► operator_adjusted
  ──► pending_client_confirm ──► confirmed ──► pending_deposit
  ──► deposit_received ──► pending_performer ──► performer_confirmed
  ──► performing ──► finished ──► pending_final_payment
  ──► final_payment_received ──► settled

任何状态可进入：
  cancelled / refunding
```

运营手动推进的状态：`pending_deposit→deposit_received`、`pending_final_payment→final_payment_received`、`finall_payment_received→settled`

### 8.2 Docker 容器

```yaml
# docker-compose.yml 核心服务
services:
  supabase-db:     # PostgreSQL
  supabase-auth:   # GoTrue
  supabase-rest:   # PostgREST
  supabase-realtime:  # WebSocket
  supabase-storage:   # 文件存储
  supabase-meta:      # 元数据
  meilisearch:         # 全文搜索
  nginx:               # 反向代理+SSL
  app:                 # 自建Node.js业务服务
```

### 8.3 CI/CD 流程

```
git push (main)
  → GitHub Actions 触发
  → 构建Docker镜像
  → 推送到 Docker Hub (or 腾讯云镜像仓库)
  → SSH登录服务器
  → docker compose pull && docker compose up -d
  → 健康检查
  → 部署完成通知
```

## 十、注意事项与风险

| 风险 | 说明 | 应对 |
|------|------|------|
| DeepSeep API不可用 | 2025年出现过大规模故障 | **备用LLM（通义千问）自动切换** |
| Supabase自托管维护 | 需要Docker+PostgreSQL运维能力 | 初期熟悉Docker Compose常用命令 |
| 微信小程序审核 | "演出"类目可能需要许可证 | **第1周就提交空白小程序测试类目** |
| 2C2G内存不足 | Supabase全套服务内存占用高 | **监控内存，必要时上线前升级2C4G** |
| NutUI兼容性 | 替换uView可能有少数组件不兼容 | **先做技术验证，保留uView回退方案** |
