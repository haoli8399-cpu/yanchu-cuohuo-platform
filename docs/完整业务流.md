# 演出撮合平台 · 完整业务流

## 四个角色

| 角色 | 定位 | 使用端 |
|------|------|--------|
| **甲方/直客** | 有演出需求的终端客户（年会、开业、婚礼等），由活动公司服务 | 小程序（第一阶段暂不开放） |
| **活动公司** | 平台的核心用户，平台的分销渠道。**需入驻审核**，可邀请同行加入 | Web端（完整工作台）+ 小程序（基础功能） |
| **演员** | 平台供给侧，演出执行者。**需入驻审核**，可邀请同行加入 | 小程序 |
| **平台运营** | 后仰团队，维护SKU+AI方案+分配演员+履约保障+结算 | Web运营后台 |

## 活动公司入驻流程（两步骤）

> 借鉴找钢网、猪八戒模式：**注册即用，交易前认证挡板。**

### Step 1：快速注册（无需审核，即刻开通）

```
活动公司打开Web端 → 点击"注册"
  → 支持两种注册方式：
     ├── **微信扫码登录**（推荐）
     │   扫码 → 授权微信头像/昵称 → 自动创建账号
     │   → 补充公司简称+服务类目 → 完成
     │
     └── 手机号注册
          输入手机号+验证码 → 设置密码
          → 填写公司简称+联系人+服务类目 → 完成
  → 完成注册 → 直接进入工作台
```

注册后的权限：
- ✅ 浏览SKU方案库
- ✅ 查看AI出方案
- ✅ 电话咨询
- ❌ 提交需求/获取报价（弹窗引导至Step 2）
- ❌ 签署合同（弹窗引导至Step 2）

### Step 2：企业认证（首次下单前挡板）

```
活动公司浏览SKU → 点击"获取报价"
  │
  ▼
弹出认证引导页（全屏弹窗）
  │ "为保障交易安全，请先完成企业认证，仅需3分钟"
  │
  ├── 填写企业信息
  │   ├── 公司全称
  │   ├── 统一社会信用代码（OCR识别）
  │   ├── 营业执照上传
  │   └── 经营地址
  │
  ├── 法人信息
  │   └── 法人身份证正反面（OCR识别）
  │
  ├── 对公账户（用于结算）
  │   ├── 开户银行
  │   ├── 开户账号
  │   └── 系统自动校验（小额打款验证）
  │
  └── 提交 → 自动校验（1-3分钟）
      ├── 通过 → 解锁全部交易功能
      └── 异常 → 转人工审核（1个工作日内）
```

认证后的权限：
- ✅ 提交需求/获取报价
- ✅ 签署合同
- ✅ 查询交易记录
- ✅ 账号显示"已认证"标识

### 邀请同行

```
已认证的活动公司 → 点击「邀请同行」
  → 生成邀请链接/海报
  → 同行点击 → 走Step 1快速注册
  → 同行认证后 → 双方获得权益（优先派单+首单优惠）
```

## 完整业务流（全角色）

### 第一阶段：SKU方案库建设（只有运营参与）

```
平台运营
  │
  ├── 创建标准化SKU
  │   ├── SKU名称：年会脱口秀豪华版
  │   ├── 适用场景：企业年会/年终总结
  │   ├── 演员画像：2名脱口秀演员
  │   │   （风格：幽默讽刺/互动强；信誉分要求≥4.0）
  │   │   （非锁定阵容，最终根据档期匹配合适演员）
  │   ├── 演出时长：45-60分钟
  │   ├── 批发价：¥5,000
  │   ├── 样片/案例：关联案例库
  │   └── 上架/下架状态
  │
  ├── 维护AI通用模板
  │   └── 当SKU库无匹配时使用
  │
  └── 维护演员库
      ├── 演员档案（资料/风格/作品）
      ├── 演员信誉分（接单率+评价+履约率）
      └── 签约状态+排他协议
```

### 第二阶段：需求产生（活动公司/甲方）

**路径A：选购SKU（用户明确知道要什么）**

```
活动公司
  │ 登录 → 浏览SKU方案列表
  │ 筛选：按类型/价格/适用场景/演员风格
  │
  ├── 选中SKU → 查看详情
  │   ├── 方案介绍+样片/案例
  │   ├── **演员画像：SKU展示的是这类演员的风格画像**
  │   │   （如：2名脱口秀演员，风格幽默互动强）
  │   │   **非锁定阵容，最终演员根据档期匹配**
  │   ├── 价格+时长
  │   └── 可勾选演员偏好（不保证）
  │
  └── 点击"获取报价"（非直接下单）
      └── 进入方案审核流程（与路径B汇合）
```

**路径B：提交需求（用户不确定要什么）**

```
活动公司/甲方
  │ 描述需求（4步7题决策树）
  │ · 活动类型 / 喜剧类型 / 人数+时长+预算
  │ · 日期+场地 / 特殊要求（定制/主持人/指定演员）
  │
  ▼
平台 AI
  │ 第一步：从SKU库匹配最接近的方案
  │ ├── 匹配成功 → 基于SKU微调（换演员/调时长/增减内容）
  │ └── 未匹配 → 从通用模板生成方案
  │
  └── 输出初始AI方案 → 进入方案审核流程
```

### 第三阶段：方案审核（双路径在此汇合）

```
无论是选购SKU还是AI出方案
  │
  ▼
平台运营人工审核+调整方案
  │ ├── **先通过微信/电话口头确认演员档期**
  │ ├── 根据实际档期匹配合适演员
  │ ├── 优先安排用户勾选的演员（不保证）
  │ ├── 调整价格/时长/内容（如有需要）
  │ └── 可手动替换演员/修改方案内容
  │
  ▼
推送给活动公司确认方案
  │ 活动公司收到通知 → 查看最终方案
  │ ├── 演员名单（确定阵容）
  │ ├── 价格明细
  │ ├── 方案说明
  │ └── 操作：
  │     ├── 确认方案 → 进入交易流程
  │     ├── 要求修改 → 退回运营重新调整
  │     └── 电话咨询 → 运营联系沟通
  │
  ▼
活动公司确认方案 → 进入交易流程
```

### 第四阶段：签约与定金确认（活动公司+平台）

```
活动公司确认方案
  │
  ▼
签约模式选择
  │ 运营在后台配置该订单的签约模式
  ├── 跳过合同 → 系统记录订单，无合同文件
  ├── 上传纸质PDF → 运营后台手动上传已签合同扫描件
  └── 系统电子签 → 在线签署（远期功能，第一阶段不用）
  │
  ▼
**定金确认（线下转账）**
  │ 活动公司通过银行转账/微信转账支付定金（50%）
  │ 支付完成后通知运营
  │
  ▼
运营在后台标记"定金已收"
  │ 系统记录：定金金额+收款方式+收款时间
  │ 订单状态推进到「已收定金」
  │
  ▼
**运营锁定演员档期**
  │ 运营在后台确认：该SKU对应的演员阵容
  │ · 可选默认演员或替换演员
  │ · 系统自动检测档期冲突
  │ · 标记演员到场时间（早于开演30-60分钟）
  │
  ▼
演员收到排期通知（短信+微信模板消息）
  │ 通知内容含：演出时间+到场时间+地址
  │
  ├── 演员确认接单 → 档期锁定
  └── 演员拒绝 → 运营重新分配
```

### 第五阶段：演出执行（全角色）

```
演出前1天
  │ 系统自动发送短信提醒给演员（含到场时间+地址）
  │ 系统发送微信模板消息给活动公司
  │
  ▼
演出当天
  │ 演员到场 → 打开小程序 → 点击**签到打卡**
  │ （位置定位+时间记录，计入信誉分）
  │
  │ 迟到/未打卡 → 信誉分扣减
  │ 准时签到 → 信誉分增加
  │
  ▼
运营跟踪演出状态
  │
  ▼
演出进行
  │ 演员到场 → 演出 → 完成
  │
  ▼
演出完成
  │ 进入48小时验收期
```

### 第六阶段：尾款确认（线下）

```
演出完成后（验收通过后）
  │
  ▼
活动公司通过银行/微信转账尾款（50%）
  │
  ▼
运营在后台标记"尾款已收"
  │ 系统记录：尾款金额+收款方式+收款时间
  │ 订单状态推进到「已收尾款」
  │
  ▼
订单完成
```

### 第七阶段：结算统计（线下打款）

> MVP阶段：平台记录应结算数据，实际打款由后仰按现有线下流程操作。

```
演出完成后48小时
  │ 验收规则：
  │ · 活动公司未提出异议 → 默认验收通过
  │ · 活动公司提出异议 → 平台运营介入处理
  │
  ▼
验收通过
  │
  ├── 系统记录应结算数据
  │   ├── 演员：待结算金额+明细（可查看，不可提现）
  │   ├── 活动公司：消费记录（可查看/导出）
  │   └── 运营后台：月度结算汇总（可导出Excel对账）
  │
  └── 线下操作
      ├── 后仰按原有方式完成实际打款
      └── 运营在系统标记"已结算"

### 第八阶段：评价与沉淀（活动公司+演员+平台）

```
演出完成
  │
  ├── 活动公司评价演员
  │   ├── 评分（1-5星）
  │   ├── 文字评价
  │   └── 该评价计入演员信誉分
  │
  ├── 平台运营给演员演出反馈
  │   ├── 现场效果评价
  │   ├── 改进建议
  │   └── 下次类似活动的建议
  │
  └── 数据沉淀
      ├── 演员：信誉分更新+案例+历史订单
      ├── 活动公司：历史采购记录+信用等级
      ├── SKU库：该SKU的成交数据+客户反馈
      └── 平台：积累行业演出经验数据
```

## 三端视图

### 活动公司端看到什么

```
1. 首页 ─── SKU方案推荐（新品/热门/按场景分类）
2. SKU列表 ─── 筛选+搜索，每个SKU显示价格+时长+适用场景
3. SKU详情 ─── 方案介绍+演员阵容+信誉分+样片+案例
4. 下单页 ─── 选日期+地址+确认
5. 需求提报 ─── 4步7题决策树
6. AI方案结果 ─── SKU匹配结果+微调方案
7. 订单列表 ─── 所有订单+状态
8. 订单详情 ─── 时间线+阵容+合同+付款入口
9. 历史记录 ─── 所有采购记录
```

### 演员端看到什么

```
1. 首页 ─── 待确认排期数 / 今日演出 / 待结算总额
2. 排期日历 ─── 月视图，可标不可用
3. 演出订单 ─── 待确认 / 已确认 / 已完成
4. 订单详情 ─── 时间地点+价格+阵容（可看同台演员）
5. 结算中心 ─── 即时到账明细（每笔展开）
6. 我的主页 ─── 职业信用档案（信誉分+评价+案例）
7. 个人中心 ─── 入驻状态/签约信息
```

### 运营后台看到什么

```
1. SKU管理 ─── 新增/编辑/上架/下架SKU
2. 需求管理 ─── 接收需求→AI出方案→审核
3. 排期分配 ─── 选演员→推送→跟踪确认
4. 演员管理 ─── 档案+信誉分+签约管理
5. 演员档期看板 ─── 全员日历视图
6. 订单监控 ─── 所有订单一览+违约预警
7. 结算中心 ─── 演员即时到账48h+活动公司尾款跟踪
8. 演员信誉分管理 ─── 规则配置+异常处理
9. 活动公司信用管理 ─── 信用分级+账期配置
10. 运营数据看板 ─── 供需健康指标
```

## 八、优化项补充

| # | 问题 | 解决方案 | 优先级 |
|:-:|------|---------|:------:|
| ① | SKU"提交需求"文案误导 | 改为**"获取报价"**，告知用户提交后运营确认最终阵容和价格 | P2 |
| ② | 活动公司确认后演员拒绝体验崩塌 | **运营口头确认演员档期后**再生成最终方案推给活动公司 | **P0** |
| ③ | 活动公司看不到方案历史版本 | 方案详情显示版本记录（v1/v2/v3），可查看历史内容 | P1 |
| ④ | 演员不知道派单来源 | 排期详情标注：**活动公司指定 / 平台风格匹配推荐**，展示匹配原因 | P1 |
| ⑤ | 演员信誉分是黑箱 | 信誉分页面展示**变动明细**（完成演出+1、拒绝-0.5、好评+0.5等） | P1 |
| ⑥ | 运营每次调整从空白开始 | **方案调整模板**：预存常用调整内容，一键应用 | P1 |
| ⑦ | 运营无法快速扫视待处理方案 | **卡片模式**：显示活动公司+预算+类型+紧急程度+已等待时长 | P1 |
| ⑧ | SKU展示形式优化 | 卡片式展示（首图+价格+标签），底栏双入口：**立即下单** / **AI定制** | **P0** |
| ⑨ | 双向评价+超时默认好评 | 演出完成48h内双方评价，互不可见待双方提交后公开，超时默认4星 | P1 |
| ⑩ | SKU加购式调整 | 基础SKU可选加购项（+演员/延时长/加主持人），实时更新总价 | P1 |
| ⑪ | 多方案并行对比 | 运营可同时推送2-3个方案（不同演员/不同价位），活动公司对比选择 | P1 |
| ⑫ | 演员到场时间+签到打卡 | 分配时标记演员到场时间（早于开演30-60分钟）。演员到场后点击**签到打卡**（位置定位），计入信誉分 | **P0** |
| ⑬ | 沉睡客户提醒 | 活动公司超过30天未下单，自动标记推送给运营主动联系 | P1 |
