# 演立方 YANLI — AI Agent 行为规则

## 核心原则

1. **技术问题直接决定，业务问题才确认** — 技术可行性的判断不许问用户，自己查文档/试错解决
2. **不许让用户改代码** — 所有修改由 Agent 完成，用户只负责确认方向和验收
3. **需求必须先查文档再行动** — 每次设计方案前必须读 PROJECT_CONTROL.md 和 PRD
4. **交付必须端到端验证** — 改完代码必须构建/部署/联调，不能只写逻辑
5. **禁止自作主张改代码** — 用户没发"开干/执行/确认"指令，不准碰任何代码文件

## 组件开发规范

1. 所有颜色引用 CSS Variable，禁止硬编码 hex
2. 字体：中文 PingFang SC / 英文 Inter / 等宽数字 JetBrains Mono
3. 每个组件必须处理 Loading / Empty / Error 三态
4. 触控目标 PC ≥ 32px，移动 ≥ 44px
5. 按钮文案按 PRD 不动（获取报价/提交需求/确认方案等）

## 品牌红线

- 禁用词：赋能/闭环/一站式/生态/沉淀/工作流/CRM/票务/厂牌/元宇宙/颠覆/连接供需
- "订单"仅后台用，对外用"成交/成单"
- 品牌名唯一："演立方"；英文 YANLI（主）/ YANLIFANG（全拼）/ YLF（缩写）
- 副标题固定："一句话需求，快速成交一场演出"
- 占位语："商演找演立方"

## 记忆规则

- 用户偏好、环境配置 → 存 memory
- 可复用的流程/方案 → 存 skill
- 任务进度、问题日志 → 不存 memory（用 session_search 回溯）
